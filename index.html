  1. Node

    1. nodeType: Node类型，常用：

      • 1: 元素节点 element

      • 2：属性节点 attr

      • 3：文本节点 text

      • 9: 文档节点 document

      • 8：注释 comments

    2. nodeName: Node名称，

      • 元素节点为标签名
      • 属性节点为属性名
      • 文本节点始终为 #text

      • 文档节点始终为 #document

        nodeName返回值始终为大写字母

    3. nodeValue: Node值，

      • 元素节点为null
      • 属性节点为属性值
      • 文本节点为文本内容
      • 文档节点为null

  2. 节点关系

    1. childNodes: 每一个节点都有一个childNodes属性，它包括节点的子节点，它是一个NodeList对象。NodeList是一个类数组对象，用于保存一组有序的节点，可以通过位置来访问这些节点。

      • 通过方括号来访问. 如：document.documentElement.childNodes[0] //
      • 通过item()方法来访问。如：document.documentElement.childNodes.item(0) //
      • length属性： 返回当前NodeList长度，不过长度是动态变化，根据DOM结构的变化而改变

    2. parentNode：每个节点都一个parentNode属性，它指向节点的父节点。包含在childNodes里面的节点都有同一个父节点

    3. previousSibling: 前一个同胞节点

    4. nextSibling: 后一个同胞节点

    5. firstNode: 第一个子节点

    6. lastNode: 最后一个子节点

    7. hasChildNodes()方法：用于判断节点是否有子节点

    8. ownerDocument: 该属性指向表示整个文档的文档节点，也就是#document

      一图解释节点关系

  3. 操作节点

    1. appendChild(): 用于向childNodes列表的末尾添加一个节点。 添加节点后，childNodes的新增节点、父节点及以前的最后一个节点的关系指针都会相应的得到更新。返回值： 新的的节点。如果传入到appnedChild（）中的节点已经是文档中的一部分。那结果就是将该节点从原来的位置转移到时新位置。因此，如果在调用appendChild()时传入了父节点的第一个子节点，那么节点就会变成父节点的最后一个子节点。

    2. insertBefore(): 可以将节点放入到childNodes列表中某个特定的位置上。这个方法接受两个参数： 要插入的节点和作为参照的节点。插入节点后，被插入的节点会变成参照节点的前一个同胞节点，同时被方法作为返回值返回。如果参照节点是null，则insertBefore()和appendChild()执行相同的操作。

    3. replaceChild(): 替换节点。接受两个参数： 要插入的节点和要替换的节点。要替换的节点将由这个方法返回并从文档树中被移除，同时由要插入的节点占据其位置。

    4. removeChild(): 移除节点。接受一个参数，要移除的节点。返回值就是要移除的节点。

    5. cloneNode(): 用于创建调用这个方法的节点的一个完全相同的副本。childNode()只接受一个布尔值参数，表示是否深复制。

      • true: 执行深复制，也就是复制节点及其整个子节点树。
      • false: 执行浅复制，即只复制节点本身。
